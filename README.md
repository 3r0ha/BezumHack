## Описание

Envelope -- это полноценная платформа для управления студией веб-разработки, закрывающая полный цикл взаимодействия с заказчиками: от приёма ТЗ и автоматического планирования проекта силами AI до сдачи релиза и еженедельной отчётности.

Платформа объединяет в себе:
- **Управление проектами** -- спринты (эпохи), задачи, Kanban-доска, Gantt-диаграмма
- **Документацию** -- иерархическая система с версионированием, ролевым доступом и inline-виджетами
- **Коммуникации** -- встроенный чат (WebSocket), видеозвонки (WebRTC), планирование встреч с голосованием за слоты
- **AI-инструменты** -- автопилот (ТЗ в план), суммаризация, оценка сложности, аналитика, матрица рисков
- **CI/CD** -- интеграция с GitLab/GitHub через webhooks, автообновление статусов задач по PR
- **Портал клиента** -- read-only дашборд для заказчика с согласованиями
- **Доски (Miro-аналог)** -- бесконечный канвас с real-time совместной работой
- **Уведомления** -- in-app + Telegram-бот

    │ TypeScript│ │ API-ключи │ │ CRUD    │ │ WebRTC      │ │ Claude   │ │ Redis PubSub│

                        │            │             │                             │

## Сущности и их связи

Платформа содержит **27 моделей** в 4 базах данных. Ниже -- все сущности и их взаимосвязи.

### Ядро (projects_db)

| Сущность | Описание | Ключевые связи |
|----------|----------|---------------|
| **Project** | Проект -- верхний уровень. Содержит название, описание, статус (`DRAFT/ACTIVE/ON_HOLD/COMPLETED/CANCELLED`), привязку к клиенту и менеджеру, дедлайн, почасовую ставку. | -> Epoch[], Task[], Document[], Meeting[], Board[], Approval[], Invoice[], Webhook[] |

| **Comment** | Комментарий к задаче. | -> Task |
| **TimeEntry** | Запись рабочего времени. Start/stop таймер или ручной ввод часов. | -> Task |

| Сущность | Описание | Ключевые связи |

## Решение болей компании

### Боль 1: Иерархия документов

**Проблема:** Документы хранятся в разрозненных папках Google Docs без привязки к задачам, спринтам и статусам. Невозможно быстро найти нужный документ, понять его актуальность или связь с конкретной задачей.

**Решение -- Docs-core:**
- **Иерархия** `Проект -> Эпоха -> Документ` -- документы структурированы по спринтам
- **Ролевой доступ** (`PUBLIC / TEAM / MANAGERS_ONLY`) -- заказчик видит только то, что ему предназначено
- **Версионирование** с changelog -- каждое сохранение создаёт новую версию, история хранится полностью
- **Inline-виджеты** -- в теле документа отображается live-статус привязанных задач с цитированием конкретных фрагментов
- **Утверждение** -- менеджер может перевести документ через workflow `DRAFT -> PENDING_REVIEW -> APPROVED -> ARCHIVED`
- **Привязка к встречам** -- версия документа помечается как принятая по итогам конкретной встречи (с AI-суммаризацией)

### Боль 2: Планирование встреч

**Проблема:** Календарь используется только для согласования даты. Результаты встречи теряются, action items не отслеживаются, связь с задачами и документами отсутствует.

**Решение:**
- **Создание из задачи** одной кнопкой -- встреча автоматически наследует участников, эпоху, документы
- **Двустороннее голосование за слоты** -- организатор предлагает несколько временных слотов, участники голосуют, система автоматически выбирает оптимальный
- **AI-суммаризация транскрипции** -- по итогам встречи AI формирует: резюме, принятые решения, action items с ответственными, открытые вопросы
- **Интеграция с документами** -- суммаризация автоматически прикрепляется к встрече и связанным документам

### Боль 3: Работа с кодом и релизы

**Проблема:** Интеграция таск-трекера с GitLab ограничена, нет связи между PR, задачами, документами и релизами.

**Решение -- CI/CD-core:**
- **Webhook-интеграция** с GitLab и GitHub -- POST `/api/cicd/webhook` принимает события от систем контроля версий
- **Автообновление статусов** задач по PR: `OPEN -> REVIEW`, `MERGED -> DONE`, `CLOSED -> TODO`
- **Привязка ветки и PR** -- задача хранит `gitBranch` и `prNumber`, статус PR обновляется автоматически
- **Git-тег -> Релиз** -- при push тега в GitLab/GitHub релиз с совпадающим `gitTag` автоматически переходит в `PUBLISHED`
- **Единые уведомления** -- все CI/CD события транслируются через общую систему уведомлений

## Ключевые фичи

### Docs-core

- Иерархия документов по проектам и эпохам с ролевым доступом (`PUBLIC / TEAM / MANAGERS_ONLY`)
- Версионирование с changelog -- полная история всех правок
- Inline-виджеты: live-статус привязанных задач прямо в теле документа
- Цитирование: при привязке задачи к документу выделяется конкретный фрагмент текста
- Связь со встречами: версия документа помечается как принятая по итогам встречи
- Согласование изменений: workflow утверждения с уведомлениями (`DRAFT -> PENDING_REVIEW -> APPROVED`)

### Kanban-core

- Drag-n-drop Kanban-доска с 5 колонками (`BACKLOG / TODO / IN_PROGRESS / REVIEW / DONE`)
- Фильтрация по приоритету (`LOW / MEDIUM / HIGH / CRITICAL`) и исполнителю
- Система зависимостей между задачами с автоматическим определением циклов
- Gantt-диаграмма с вычислением критического пути
- Трекинг времени (start/stop таймер) с привязкой к задачам
- Комментарии к задачам

### Коммуникация

- **Встроенный чат** с WebSocket (Socket.IO) -- обмен сообщениями в реальном времени
- **Видеозвонки** (WebRTC, mesh topology) -- полноценный аналог Google Meet
- **Публичный доступ** по ссылке `/meet/:id` -- гости могут подключиться без авторизации
- **Перевод сообщений** через AI -- встроенный переводчик с автоопределением языка
- **Прикрепление файлов** -- загрузка и скачивание файлов в чате (до 50 МБ)

### AI-инструменты

Все AI-эндпоинты работают с **YandexGPT** или **Anthropic Claude** (настраивается через `.env`). Без ключей -- возвращают mock-ответы для демонстрации.

| Инструмент | Описание |
|-----------|----------|
| **Автопилот** | Загрузка ТЗ (текст или файл) -> полный план проекта: фазы, задачи с зависимостями, оценки, риски, рекомендации по стеку |
| **Суммаризация ТЗ** | Краткое резюме технического задания с выделением ключевых требований, стека, сроков и рисков |
| **Декомпозиция задач** | Автоматическая разбивка задач на подзадачи с определением зависимостей и приоритетов |
| **Оценка сложности** | AI-оценка трудозатрат задачи (одиночная или пакетная) с reasoning и предложением подзадач |
| **Аналитика проекта** | Health score, точность оценок, узкие места, velocity, прогноз завершения к дедлайну |
| **Еженедельный отчёт** | Клиентский отчёт на понятном языке: прогресс, риски, метрики, что нужно от заказчика |
| **Матрица рисков** | Идентификация рисков проекта с оценкой вероятности, импакта, категории и мер митигации |
| **Суммаризация встреч** | Транскрипция -> резюме, решения, action items с ответственными, открытые вопросы |

### Интеграции

- **CI/CD**: GitLab/GitHub webhooks (`POST /api/cicd/webhook`), автообновление статусов задач по MR/PR, git-тег -> публикация релиза
- **Telegram**: бот `@envelope_notifybot` для push-уведомлений. Привязка аккаунта через настройки
- **API-ключи**: полное REST API с авторизацией через Bearer-токен или API-ключ (генерация в `/settings/api`)
- **Webhooks**: настраиваемые исходящие webhooks на события платформы (с подписью запросов)
- **Документы <-> Задачи**: прямое цитирование, автопривязка, inline-виджеты с live-статусом

### Портал клиента

- Read-only дашборд для заказчика с прогрессом проекта в реальном времени
- Просмотр задач (без редактирования)
- Документы только с видимостью `PUBLIC`
- Система согласований (`Approval`) -- заказчик получает запросы и может одобрить/отклонить
- Ограниченный доступ -- клиент видит только свои проекты

### Доски (аналог Miro)

- Бесконечный канвас с поддержкой pan/zoom
- Элементы: стикеры, фигуры, текст, линии, рисование от руки
- Настройка цвета, размера, поворота, z-index
- Real-time совместная работа (Socket.IO)
- Привязка к проектам

### Биллинг

- Создание счетов с позициями (часы, ставка, сумма)
- Привязка позиций к конкретным задачам
- Workflow: `DRAFT -> ISSUED -> PAID / OVERDUE / CANCELLED`
- Почасовая ставка проекта

### Администрирование

- Управление пользователями: создание, назначение ролей, удаление
- Назначение на проекты (клиент, менеджер)
- Спринты/Эпохи: создание, управление целями, автосинхронизация статуса из задач
- Аудит-лог всех действий пользователей

## Система уведомлений

Единая система уведомлений охватывает все события платформы и доставляет их через два канала: **in-app** (Socket.IO WebSocket) и **Telegram** (бот).

| Событие | Тип уведомления |
|---------|----------------|
| Назначение задачи | `TASK_ASSIGNED` |
| Смена статуса задачи | `TASK_STATUS_CHANGED` |
| Комментарий к задаче | `TASK_COMMENT` |
| Разблокировка задачи | `BLOCKER_RESOLVED` |
| Приближение дедлайна | `DEADLINE_APPROACHING` |

## Запуск

### Требования

- Docker и Docker Compose
- (Опционально) API-ключ YandexGPT или Anthropic Claude для AI-функций

### Быстрый старт

```bash
git clone https://github.com/3r0ha/BezumHack.git antihype-platform
cd antihype-platform

# Скопировать и настроить переменные окружения
cp .env.example .env

# Запуск в dev-режиме (с hot-reload)
make dev
# или напрямую:
docker compose -f docker-compose.yml -f docker-compose.dev.yml up --build

# Запуск в production-режиме
make prod
# или:
docker compose up --build -d
```

Платформа будет доступна по адресу **http://localhost:80**.

### Инициализация данных

```bash
# Применить схемы БД
make db-push

# Заполнить демо-данными (пользователи, проекты, задачи)
make seed

# Полный сброс + демо-данные
make reset
```

### Настройка AI (опционально)

AI-сервис работает без ключей, возвращая mock-ответы для демонстрации. Для реальных AI-функций настройте `.env`:

```env
# Вариант 1: YandexGPT
AI_PROVIDER=yandex
YANDEX_API_KEY=...
YANDEX_FOLDER_ID=...
YANDEX_MODEL=yandexgpt/latest

# Вариант 2: Anthropic Claude
AI_PROVIDER=anthropic
AI_API_KEY=sk-ant-...
AI_MODEL=claude-sonnet-4-20250514
```

### Настройка Telegram-бота (опционально)

```env
TELEGRAM_BOT_TOKEN=your-bot-token
```

## API

### Аутентификация

Все запросы к API авторизуются через:
- **JWT Bearer Token** -- `Authorization: Bearer <token>` (получается через `POST /api/auth/login`)
- **API Key** -- `Authorization: Bearer <api-key>` (генерируется в `/settings/api`)

### Основные эндпоинты

```
# Аутентификация
POST   /api/auth/login                     # Вход
POST   /api/auth/register                  # Регистрация
GET    /api/auth/me                        # Текущий пользователь
GET    /api/auth/users                     # Список пользователей

# Проекты
GET    /api/projects/projects              # Список проектов
POST   /api/projects/projects              # Создать проект
GET    /api/projects/projects/:id          # Детали проекта

# Задачи
GET    /api/projects/tasks/project/:id     # Задачи проекта
POST   /api/projects/tasks                 # Создать задачу
PATCH  /api/projects/tasks/:id             # Обновить задачу

# Документы (Docs-core)
GET    /api/projects/documents/project/:id # Документы проекта
POST   /api/projects/documents             # Создать документ
PATCH  /api/projects/documents/:id         # Обновить (новая версия)
POST   /api/projects/documents/:id/approve # Утвердить

# Встречи
GET    /api/projects/meetings/project/:id  # Встречи проекта
POST   /api/projects/meetings              # Создать встречу
POST   /api/projects/meetings/:id/vote     # Проголосовать за слот
POST   /api/projects/meetings/:id/summarize # AI-суммаризация

# AI
POST   /api/ai/autopilot                  # ТЗ -> план проекта
POST   /api/ai/summarize/                 # Суммаризация текста
POST   /api/ai/estimate/                  # Оценка сложности
POST   /api/ai/analytics                  # Аналитика проекта
POST   /api/ai/weekly-report              # Еженедельный отчёт
POST   /api/ai/risk-matrix                # Матрица рисков
POST   /api/ai/translate/                 # Перевод
POST   /api/ai/summarize/meeting          # Суммаризация встречи

# CI/CD
POST   /api/cicd/webhook                  # GitLab/GitHub webhook

# Чат
GET    /api/chat/conversations             # Список бесед
POST   /api/chat/messages                  # Отправить сообщение
ws://  /socket.io/                         # WebSocket (real-time)

# Уведомления
GET    /api/notifications/                 # Список уведомлений
ws://  /notifications-ws/                  # WebSocket (push)
```

Полная документация API доступна после авторизации: **Настройки -> API** (`/settings/api`).
